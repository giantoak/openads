Quick Start – X-HT Prototype

Using Web Data to Find High Risk Individuals and Detect Organized Crime

Human Trafficking, especially for the commercial sex trade, is a line of business with significant web presence to attract customers.  X-HT operates on ads that are scraped from the websites.  Each ad is a row or record in the database.  The data repository now contains several hundred thousands of ads.  Data is cleaned, normalized and refined.  Then it is grouped by entity.  By grouping ads that are about the same entity together, it facilitates the detection of behavior over time and space.  Entities can be individuals, organizations or locations.  Ads are grouped by “clustering” similar records together into one group.

The Data

  • Go to https://ocweb.istresearch.com/openads/chart.html to look at the available fields and a frequency chart of their values.
  • Each record has name, email, region, otherads (url), body (free text description), bust, location, build, title, height, rate, description, age, kitty, outcall, availability, tags, keywords, phone numbers, incall, eye color, websites, image alt, source, post timestamp, hair color, ethnicity and several additional administrative fields.
  • The fields available vary with data source. 
  • These are the fields currently typically available: region, otherads (url), body (free text description), location, title, age, phone numbers, websites, and source.

How Clustering Works

  • An order N algorithm is used that takes just one pass through the data.  The clusters or groups are pre-computed for simple searches.  In advanced searches, users can define their own clusters.
  • Records are grouped into "organizations" if one of phone #, email or website matches (in that order).
  • Records in "organizations" are further separated into sub-groups representing "individuals" by matching on age and ethnicity.
  • Records in "locations" are grouped if the "region" field matches.

How Search Works

  • Any number typed into the search field is interpreted as a phone number.  Only the phone field is searched.
  • For any characters (string) typed, if it can be, it will be interpreted as a location.  Only the region field is searched.
  • Otherwise the search is done on the free text field (i.e. body, title, email, website).
  • “Quotes” are not needed.  An AND is done on all words typed, and all words must be found together.

Try These Searches

  • “newly arrived”, “in town”, “really young”, a phone number (e.g. 5594589508), or a partial phone number

Core Workflow

There are two modes of operation: simple and advanced.  The following describes the use of the simple mode.

  1. Enter a tip (phone number, email address or location) into the search box.
  2. Select how the results should be aggregated (individuals, organizations or locations).
  3. Click the search button.
  4. The top twenty results are shown in the center ring of the graph window.  Additional entities sharing key attributes with the results are automatically added.  Colored links show how the entities are related (e.g. phone, email).
  5. Use the arrows at the top to page through results.
  6. Click to select an entity (i.e. node) to see the records comprising that entity.  The records are shown in multiple facets (movement over time, geospatial, word cloud, attribute viewer, table of records).
  7. Individual ads can be accessed from the table.  You can search on just those records to, for example, find exactly those records that matched your original search term by typing in the table search box.

Understanding Each View

Graph

The graph window shows three concentric rings.  The inner ring is the search results.  The next ring is the entities related (by email, phone or website) to the search results.  The outer ring is the entities one further step out.

Links

In the simple mode, links between nodes are drawn automatically when two nodes share a phone #, email or website.

Table

Use the search field on the table view to look for specific records that were returned on your string search.  For example if you searched on "newly arrived" and found 10 nodes, and then selected one of the nodes to see its records.  In the table view you can search to see which one of those specific records contains "newly arrived".

Movement and Map

The Movement and Map view provide a way to visually detect if an individual seems to be moving locations frequently over time.

  • The Movement view plots each ad record for the selected entity according to location and time the ad was posted.
  • The Map view plots the location of each ad record for the selected entity.  The circle size represents the number of ads posted for that location.  Selecting a circle on the Map will also show the corresponding ads in the Table view.

Word Cloud

·         The most frequent words in the selected ads are displayed. Font size encodes term frequency.

Cluster Attributes

  • The Attributes view shows the distribution of the values for the fields in the entity (i.e. cluster) selected.
  • For an entity representing an organization, multiple values for attributes such as ethnicity suggests multiple individuals are involved.
  • Or for an entity representing an individual, multiple values for location suggests movement (which can be confirmed using the Movement and Map views).
  • Click on the '+' to expand a field to see the actual values.

Hints and Warnings

  • Chrome is the fastest browser to use.  Firefox also works reasonably well.
  • The first query in the simple UI is slower (e.g. 25 secs) but then a cache is made.  Subsequent queries are faster (e.g. 5 secs).
  • If a result set is too messy when searching on people clusters, try searching on organization clusters.  There should be fewer nodes returned.
  • The system will be brought down daily for maintenance at 100 am PST for approximately an hour.

Known Issues and Bugs

  • The JTable view in the bottom panel has misaligned column titles due to a Chrome bug.  Firefox should be ok.
  • "Age" and "ethnicity" fields are often incomplete resulting in lower quality "individual" clusters
  • By defining a location using "region", very large clusters are formed.  "Region" is NY for instance with one node with many thousands of records.  And then there will be many thousands of email/phone/website links drawn from that single big node.  "Region" is not precise enough location.  Until we have a more precise location attribute, clustering by location is disabled.
